Getting Started

XUI是携程商旅前端公共样式库，旨在统一公用页面元素和规范代码风格，达到代码复用，减少重复设计；没有最好的规则，只有最适合的规则，不管团队成员多少，代码都应同出一门。

样式表引用方式，直接引用全模块压缩模式base.css(online目前使用corphd_niv.css)，或者分别使用各模块代码

代码规范

命名规则

项目命名

项目名采用小写英文单词单数形式，国际版加下划线，例如国内机票flight、国际机票flight_int；单词过长不易表达的项目用英文大写缩写，例如短信邮件授权项目SME；尽量简洁易懂，避免用中文拼音

目录名

采用英文单词命名。例如

    + project
    ---_src /* 预编译源码文件 */
    ---_include /* 功能引用代码文件 */
    ---css /* 样式表文件 */
    ---images /* 图片文件 */
    ---js /* javascript文件 */

HTML/PHP命名

按照页面属性或业务流程命名，以英文单词单数形式，英文版加下划线，例如中文首页index.php、英文首页index_en.php

CSS命名

采用小写英文单词加下划线组合命名，大型项目同当前页面命名规则，如index.css,searchlist.css；小型项目按项目或者业务名命名，如cms.css；英文版添加下划线，如index_en.css

JavaScript命名

采用小写英文单词命名，多单词时采用单驼峰式命名，如index.js,myAccount.js

HTML

语法

  • 嵌套缩进使用四个空格的soft tabs——保证代码在各个环境下显示一致，sublime编辑器设置tab_size为4个空格即可。
  • 不要在自动闭合标签结尾处使用斜线 - HTML5 规范中指出他们是可选的，如<input value="">、<image src="">、<hr>。
  • 不要忽略可选的关闭标签，如</li>、</body>。

HTML5 doctype

在每个页面启用html5标准的文档模式，尽可能使每个浏览器展示一致，<!DOCTYPE html>。

虽然doctype大小写都可以，但是按照惯例，最好按照xhtml的惯例大写。

字符编码

以html5标准声明页面的字符编码，让浏览器正确渲染网页内容，<meta charset="UTF-8">。

浏览器兼容模式

在元标签中声明IE兼容模式为最新版本、双核浏览器以Chrome内核渲染
<meta http-equiv="X-UA-Compatible" content="IE=edge,chrome=1">

CSS

语法

  • 嵌套缩进使用四个空格的soft tabs。
  • css除prefix属性外，所有声明放在一行。
  • 为了代码的易读性，在每个声明的左括号前增加一个空格。
  • 每条声明 : 和；后应该插入一个空格。
  • 所有声明应该以分号结尾。虽然最后一条声明后的分号是可选的，但是如果没有他，你的代码会更容易出错。
  • sass每条声明只占一行。
  • sass声明块的右括号另起一行。
  • 不要在颜色值中增加空格rgb() rgba() hsl() hsla()，并且不要带有取值前面不必要的0(比如，用.5替代0.5)。
  • 不要为0指明单位，比如用margin: 0;而不是margin: 0px;。
  • 颜色值尽量使用短的十六进制数值，例如使用#fff替代#ffffff。
 最佳实例：
    /* good css */
    .selector { position: relative; top: 0; left: 10px; }
    .selector { position: relative; top: 0; left: 10px; margin: 10px; padding: 5px;
        -webkit-box-shadow: 2px 2px 3px rgba(0,0,0,.5); 
           -moz-box-shadow: 2px 2px 3px rgba(0,0,0,.5); 
            -ms-box-shadow: 2px 2px 3px rgba(0,0,0,.5); 
    }
    /* good sass */
    .sass-selector { 
        position: relative; 
        top: 0; 
        left: 10px; 
    }

声明顺序

相关的属性声明应该以下面的顺序分组处理：

  1. 1.布局定位Position
  2. 2.合模型Box model
  3. 3.排版Typographic
  4. 4.外观Visual

Class命名

  • 保持Class命名全为小写，可以使用中横线(不要使用下划线 class_name 和单驼峰式 className命名)。
  • 避免过度简写。如按钮定义为.s
  • Class的命名应该尽量短，也要尽量明确。如将按钮定义为.btn
  • 使用有意义的英文名称；使用结构化或者作用目标相关，而不是抽象的名称。
  • 命名时使用最近的父节点或者父class作为前缀。

代码注释

  • 单行注释，内容与星号保持一个空格。
  • 多行注释，内容与星号保持一个空格，除首行外星号前保持一个空格。
  • 代码块注释，前后用大写S和E封闭。
单行注释
/* block style */
.block { display: block; }

多行注释
/**
 * Components section heading
 * Question description here some context···
 */ 
.block { display: block; }

代码块A注释
/* module A S */
.moduleA { margin: 10px; padding: 5px; }
.moduleA  .module-hd { margin: 10px; padding: 5px; }
.moduleA  .module-bd { margin: 10px; padding: 5px; }
/* module A E */

选择器

  • 使用classes而不是直接使用元素标签来声明样式。
  • 尽量减少选择器长度，且组合选择器层级不超过三级。
  • 谨慎在经常出现的组件中使用低效率选择器，如[class^="..."]。
/* Bad example */
div p { display: block; }
.page-container .container-head .container-list-item  .container-item-one { ··· }

/* Good example */
.module { margin: 10px; padding: 5px; }
.module .module-hd { margin: 10px; padding: 5px; }
.module .module-bd .avator { margin: 10px; padding: 5px; }

代码组织

  • 现阶段(未实现前端自动化)建议以每个页面为单位组织代码，相互之间降低耦合。
  • 尽量以模块化组件的方式组织代码，考虑复用性，降低耦合粒度。
  • 制定一个一致的注释层级结构。
  • 使用一致的空白来分割代码块，更利于查看大型的文件。

编辑器配置

根据以下的设置来配置你的编辑器，来避免常见的代码不一致和丑陋的 diffs。

  • 使用四个空格的 soft-tabs
  • 在保存时删除尾部的空白字符。
  • 设置文件编码为 UTF-8。
  • 在文件结尾添加一个空白行。

参考：sublime_text HTMLBeautify

浏览器分级

  • A级：桌面现代浏览器，包含IE9+/Chrome/Safari/Firefox/Opera/Webkit内核系列
  • B级：IE8
  • C级：IE7

CSS3优先，兼容优先级：A > B > C
